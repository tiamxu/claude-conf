# 代码优化

## 职责定位

**输入**：待优化代码文件 → **输出**：优化后代码 + 优化说明

**优化范围**：代码简化、性能优化、函数拆分、重复代码抽象、错误处理优化

**定位**：专注于代码质量和性能优化，不做架构层面的重构

---

## 两种模式

| 模式 | 命令 | 说明 |
|---|---|---|
| 增量优化 | `/code-optimizer` | 优化用户提供的文件 |
| 全量优化 | `/code-optimizer full` | 扫描并优化整个项目所有代码文件 |

**增量优化**：用户直接说明要优化的文件，AI 读取并优化

**全量优化**：AI 直接扫描项目目录下所有 `.go` 文件，找出可优化的问题

---

## 🔍 优化维度

### 1. 代码简化

- [ ] 简化嵌套层级（提前返回/提前出错）
- [ ] 合并相似逻辑（相同模式抽象）
- [ ] 使用简洁语法（短变量声明、range 代替 for）
- [ ] 删除死代码（未使用的变量、函数、import）
- [ ] 简化条件表达式（使用 Bool 函数封装复杂条件）

### 2. 重复代码抽象

- [ ] 识别重复代码模式（超过 3 处）
- [ ] 抽象为公共函数
- [ ] 提取重复字符串为常量
- [ ] 提取工具函数到 utils/
- [ ] 识别魔法数字/字符串，提取为命名常量

### 3. 函数拆分

- [ ] 过长函数（> 80 行）拆分
- [ ] 多职责函数拆分（职责分离原则）
- [ ] 提取独立逻辑为子函数
- [ ] 参数过多时封装为结构体或选项模式
- [ ] 识别"上帝函数"并拆分

### 4. 性能优化

**Go 语言**
- [ ] 循环内拼接字符串 → `strings.Builder`
- [ ] 已知容量的切片预分配 `make([]T, 0, cap)`
- [ ] 避免在循环内分配内存（提至循环外）
- [ ] `defer` 释放资源（文件、连接）
- [ ] 批量操作代替循环单条操作
- [ ] 减少 Goroutine 泄漏（确保 channel 关闭）
- [ ] 添加 context 超时控制
- [ ] 正确使用 sync.Pool 复用对象

**Go SQL**
- [ ] 使用参数化查询，禁止字符串拼接 SQL
- [ ] 批量查询代替循环单条查询
- [ ] 使用只读事务（`ReadOnly()`）
- [ ] 合理使用索引

### 5. 错误处理优化

- [ ] 错误正确包装 `fmt.Errorf("操作失败: %w", err)`
- [ ] 禁止丢弃 `err`（使用 `_ = err` 前需注释原因）
- [ ] 错误类型细分（自定义错误类型）
- [ ] 统一错误返回格式（`pkg/e` 统一错误码）
- [ ] 关键操作日志记录错误

### 6. 并发安全

- [ ] 竞态条件检查（`go run -race`）
- [ ] 正确使用 mutex（读写锁优化读多写少场景）
- [ ] 避免共享变量，使用 channel 传递
- [ ] 确保 channel 关闭（只关闭一次）
- [ ] 同步点正确使用（`sync.WaitGroup`）

### 7. 可读性优化

- [ ] 变量命名语义化
- [ ] 添加关键注释（复杂逻辑）
- [ ] 保持代码风格一致
- [ ] 导出/非导出正确使用

## ⚡ 执行规则

1. **不使用 git 命令**
2. 增量优化：直接读取用户提供的文件
3. 全量优化：直接扫描项目目录下所有 `.go` 文件
4. 先理解代码逻辑，再进行优化
5. 每个优化项必须给出优化前/后对比
6. 优化后代码必须保持功能不变
7. 性能优化需说明性能提升点（估算）
8. 代码变化需说明减少/增加的代码行数
9. 涉及架构调整时需提前确认
10. 优先级：性能问题（P0）> 代码质量（P1）> 可读性（P2）
