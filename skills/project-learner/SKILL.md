# 项目学习

## 职责定位

**目标**：30 分钟内掌握项目架构，可承接开发任务

**输入**：无 或指定路径 → **输出**：项目认知报告

---

## 两种模式

| 模式 | 命令 | 说明 |
|---|---|---|
| 快速了解 | `/project-learner` | 30分钟掌握核心架构，可承接开发 |
| 深度学习 | `/project-learner deep` | 深入学习完整项目，产出详细项目手册 |

---

## 🔍 学习流程

### 第一步：读取项目规范（如有）

- [ ] CLAUDE.md（仅作参考，可能过时）
- [ ] README.md（仅作参考，可能过时）
- [ ] go.mod / package.json（技术栈以这个为准）

### 第二步：核心架构扫描

| 文件/目录 | 职责 |
|---|---|
| main.go / main.ts / App.vue | 程序入口 |
| config/ | 配置加载 |
| routes/ / router/ | 路由定义 |
| middleware/ | 中间件 |
| api/ | 接口层 |
| service/ | 业务逻辑 |
| repo/ / dao/ | 数据访问 |
| model/ | 数据模型 |
| types/ / dto/ | 请求响应结构 |

### 第三步：关键规范提取

- [ ] 分层规范（各层职责）
- [ ] 命名规范（文件、结构体、函数）
- [ ] 错误处理（统一错误码）
- [ ] 鉴权方式（JWT/Token）
- [ ] 日志规范（格式、级别）

### 第四步：接口清单梳理

| 模块 | 路径 | 方法 | 说明 | 鉴权 |
|---|---|---|---|---|
| 用户 | /api/v1/users | GET | 获取用户列表 | JWT |
| ... | ... | ... | ... | ... |

### 第五步：业务流程还原

- [ ] 核心请求链路（入口→数据库）
- [ ] 关键业务流程（下单流程、支付流程等）
- [ ] 异步处理（MQ/异步任务）

## 📝 输出格式

```markdown
# 项目认知报告

## 基本信息
| 项目 | 内容 |
|---|---|
| 项目名称 | ecommerce |
| 技术栈 | Go 1.25 + Gin + MySQL |
| 构建状态 | ✅ 编译成功 |

## 系统架构
```
[前端] → [API Gateway] → [BFF] → [Service] → [Repo] → [MySQL]
                              ↓
                         [Redis Cache]
```

## 技术栈
- 语言：Go 1.25
- 框架：Gin Web Framework
- 数据库：MySQL 5.7
- 缓存：Redis
- 基础库：kit/sql、kit/log

## 项目规范

### 分层规范
- api/：参数解析、调用 Service、返回响应
- service/：业务逻辑，不写 SQL
- repo/：数据访问，不写业务逻辑

### 命名规范
- 文件：小写蛇形（user_service.go）
- 结构体：大写驼峰（UserService）
- 函数：New 前缀构造器

### 错误处理
- 使用 pkg/e 统一错误码
- Service 层错误包装：fmt.Errorf("操作失败: %w", err)

### 鉴权
- JWT Token
- 路径前缀：/api 需要鉴权

## 核心接口

| 模块 | 接口数 | 说明 |
|---|---|---|
| 用户 | 5 | 登录/注册/个人信息 |
| 商品 | 8 | CRUD + 上下架 |
| 订单 | 6 | 创建/支付/查询 |

## 核心业务流程

### 下单流程
1. 用户选择商品 → 加入购物车
2. 结算页确认 → 创建订单
3. 选择支付方式 → 发起支付
4. 支付成功 → 更新订单状态 → 库存扣减

### 关键判断节点
- 库存不足 → 提示用户
- 支付超时 → 关闭订单
- 管理员审核 → 状态流转

## 环境配置

| 环境 | 配置位置 |
|---|---|
| dev | config/config-dev.yaml |
| test | config/config-test.yaml |
| prod | config/config-prod.yaml |

## 部署方式
- 后端：`go build -o main`
- 前端：`npm run build`
- Docker：`docker-compose up`

## 独立开发就绪

| 模块 | 状态 | 可承接任务 |
|---|---|---|
| 用户模块 | ✅ 已掌握 | 用户 CRUD、登录注册 |
| 商品模块 | ✅ 已掌握 | 商品管理、上下架 |
| 订单模块 | ⚠️ 待深入 | 订单流程优化 |
```

## ⚡ 执行规则

### 快速了解模式
1. 按顺序执行，不要跳步
2. 核心是理解架构和接口，而非深入代码细节
3. 输出报告要聚焦"能否承接开发任务"
4. 不确定的地方标注"待确认"
5. 完成后给出"独立开发就绪"结论

### 深度学习模式
1. 深入学习每个模块，不留死角
2. 代码走读，理解核心逻辑
3. 输出完整项目手册
4. 标注已掌握模块和待深入模块
